# GitHub als Tool voor Data Stewardship en Reproduceerbaarheid

### ŌØż Hoe is GitHub gebruikt voor het projectbeheer?
Binnen dit project is GitHub gebruikt als de centrale online omgeving (*repository*) waar alle onderdelen van het onderzoek ŌĆö van ruwe data tot de uiteindelijke R-scripts ŌĆö op ├®├®n veilige plek zijn samengebracht. Als data steward zorgde dit ervoor dat het project niet lokaal op een computer bleef staan, maar direct toegankelijk en back-up-veilig was gecentraliseerd in de cloud.

---

### ŌØż. Hoe draagt de GitHub-pagina bij aan de reproduceerbaarheid van de data?
De GitHub-pagina is een verzamelplaats voor alles wat de eigenaar heeft gemaakt en gebruikt. Een externe onderzoeker kan de volledige repository in ├®├®n keer downloaden (*clonen*). Omdat de exacte R-scripts, de gebruikte versies van de data en de instellingen hier online staan, kan iemand anders de code runnen en exact dezelfde resultaten en visualisaties (zoals de volcano plot en KEGG-pathways) ook verkrijgen. Wat we ook bij `Data_Stewardship.md` over hebben gehad, met openabre data. Om zo *open science* te bevorderen.

### ŌØż. Is alles makkelijk terug te vinden en te gebruiken voor een ander & Hoe is er gezorgd voor een duidelijke structuur in de repository?
Ja, dankzij de `README.md` op de startpagina kan een extern persoon direct bezig. In deze README staat een duidelijke handleiding (Materiaal en Methode) die uitlegt in welke volgorde de scripts gedraaid moeten worden en welke softwarepakketten nodig zijn. Zodat deze extern persoon, zelf minder voorbereidingen hoeft uit te voeren om bezig te gaan met het onderzoek.
Voor de rest is in github alles ook verdeeld in mappen (overzichtelijkheid), zodat het snel te vinden is wat nodig is en daardoor ook een duidelijk structuur. Ook is er gebruik gemaakt van titels om bij het scannen van de Github snel te vinden wat nodig is.

### ŌØż. Hoe is er concreet gewerkt met versiebeheer?
Github maakt gebruik van *commits* elke ker als je wat aanpast en het opslaat wordt deze versie opgeslagen, zodat alles bewaard blijft van alle aanpassingen. Zodien nodig, kan er altijd terug worden gekeken naar de eerdere versies en de aanpassingen die zich erin plaats hebben gevonden.
